## рҹ“Ӣ Overview

**SmartNGO** is a blockchain-backed transparent fund management platform designed for Non-Governmental Organizations (NGOs) and non-profit organizations worldwide. The system leverages blockchain technology to ensure complete transparency, accountability, and trust in donation management without requiring real cryptocurrency transactions.

### рҹҢҹ Why SmartNGO?

Traditional NGO donation systems face critical challenges:

- **Lack of Transparency**: Donors don't know how their funds are utilized
- **Trust Issues**: No way to verify fund allocation
- **Manual Processes**: Prone to errors and mismanagement
- **No Real-Time Tracking**: Delayed or no updates on impact

SmartNGO solves these problems by providing:

- рҹ”— **Blockchain-Backed Transparency Ledger**: Every transaction recorded immutably
- вҡЎ **Smart Contracts for Automated Fund Tracking**: Trustless, automated operations
- рҹҺҜ **Role-Based Access Control**: Donor / NGO Manager / Platform Administrator
- рҹ“Ҡ **Real-Time Monitoring Dashboard**: Live updates on donations and utilization
- рҹ“Ҳ **Financial Reports & Impact Analytics**: Comprehensive reporting tools
- рҹ”җ **JWT-Based Secure Authentication**: Industry-standard security
- рҹ’і **Simulated Payment Gateway**: No real crypto required - perfect for demos!
- рҹ“Ў **WebSocket Real-Time Updates**: Instant notifications
- рҹ§ҫ **Immutable Donation Logs**: Permanent, tamper-proof records

---

## рҹ’Ў What SmartNGO Does

SmartNGO is a **Blockchain-Based Transparent Fund Management System** that enables:

1. **For Donors**:
   - Securely donate to verified NGOs using simulated blockchain transactions
   - Track donations in real-time
   - View detailed reports on fund utilization
   - Receive instant digital receipts

2. **For NGO Managers**:
   - Register and manage organization profiles
   - Monitor incoming donations
   - Generate weekly and monthly financial reports
   - Request fund withdrawals with transparent purpose tracking
   - Showcase projects and impact metrics

3. **For Platform Administrators**:
   - Verify and approve NGO registrations
   - Monitor platform-wide transactions
   - Ensure compliance and transparency
   - Manage system integrity

### рҹҺҜ Focus Areas

SmartNGO supports all types of NGOs and charitable organizations:

- рҹҺ“ **Education**: Schools, scholarship programs, literacy initiatives
- рҹҸҘ **Healthcare**: Medical camps, hospitals, health awareness
- рҹҢҚ **Environment**: Conservation, climate action, sustainability
- рҹҶҳ **Disaster Relief**: Emergency response, rehabilitation
- рҹҗҫ **Animal Welfare**: Shelters, wildlife protection
- рҹ‘Ҙ **Community Development**: Infrastructure, livelihood programs
- вҡ–пёҸ **Human Rights**: Advocacy, legal aid, empowerment

## рҹ’і Simulated Blockchain Payment Model

### вҡ пёҸ CRITICAL: This is NOT Real Cryptocurrency

SmartNGO uses a **simulated blockchain payment system** that:

- вң… Runs on **testnet networks only** (Polygon Amoy, Ethereum Sepolia)
- вң… Uses **test MATIC/ETH** with no real-world value
- вң… Generates **real transaction hashes** on testnet
- вң… Provides **full blockchain transparency** without financial risk
- вң… Perfect for **demonstrations, academic projects, and MVPs**

### How It Works

```javascript
// Donation transaction (SIMULATED)
const tx = await ngoFundContract.donateToNGO(
  ngoWalletAddress,
  \"Education Program\",  // Purpose
  { value: ethers.parseEther(\"0.01\") }  // Testnet MATIC
);

// Transaction is recorded on blockchain (testnet)
const receipt = await tx.wait();
console.log(\"Transaction Hash:\", receipt.hash);  // Real testnet hash

// Backend stores the record
await Transaction.create({
  txHash: receipt.hash,
  amount: 0.01,
  isSimulated: true,  // Clearly marked
  network: \"polygon-amoy\"
});
```

## рҹҡҖ Getting Started

### Prerequisites

Before you begin, ensure you have:

- **Node.js** (v18 or higher)
- **npm** or **yarn** package manager
- **MongoDB** (local or cloud instance like MongoDB Atlas)
- **MetaMask** browser extension
- **Git**

### Installation

#### 1пёҸвғЈ Clone the Repository

```bash
git clone https://github.com/yourusername/smartngo.git
cd smartngo
```

#### 2пёҸвғЈ Backend Setup

```bash
cd backend

# Install dependencies
npm install

# OR using yarn
yarn install
```

#### 3пёҸвғЈ Frontend Setup

```bash
cd ../frontend

# Install dependencies
npm install

# OR using yarn
yarn install
```

### Configuration

#### Backend Environment Variables

Create `.env` file in `/backend` directory:

```env
# Server Configuration
PORT=5500
NODE_ENV=development

# MongoDB
MONGODB_URI=mongodb://localhost:27017/smartngo
# OR use MongoDB Atlas
# MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/smartngo

# JWT Authentication
ACCESS_TOKEN_SECRET=your_access_token_secret_here_min_32_chars
ACCESS_TOKEN_EXPIRY=1d
REFRESH_TOKEN_SECRET=your_refresh_token_secret_here_min_32_chars
REFRESH_TOKEN_EXPIRY=7d

# Cloudinary (for image uploads)
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret

# Email Configuration (NodeMailer)
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USER=your_email@gmail.com
EMAIL_PASS=your_app_password

# Blockchain Configuration (TESTNET ONLY)
AMOY_RPC_URL=https://rpc-amoy.polygon.technology/
PRIVATE_KEY=your_wallet_private_key_without_0x
POLYGONSCAN_API_KEY=your_polygonscan_api_key

# Optional: Ethereum Sepolia
SEPOLIA_RPC_URL=https://rpc.sepolia.org
ETHERSCAN_API_KEY=your_etherscan_api_key

# NGO Wallet for Testing
NGO_WALLET_ADDRESS=0xYourNGOWalletAddress
```

#### Frontend Environment Variables

Create `.env.local` file in `/frontend` directory:

```env
# Backend API URL
NEXT_PUBLIC_API_URL=http://localhost:5500

# WebSocket URL
NEXT_PUBLIC_WS_URL=ws://localhost:5050

# Blockchain Network (Testnet)
NEXT_PUBLIC_CHAIN_ID=80002
NEXT_PUBLIC_CHAIN_NAME=Polygon Amoy Testnet
NEXT_PUBLIC_RPC_URL=https://rpc-amoy.polygon.technology/

# Smart Contract Addresses (Update after deployment)
NEXT_PUBLIC_NGO_REGISTRY_ADDRESS=
NEXT_PUBLIC_NGO_FUND_ADDRESS=
```

### Running the Application

#### 1пёҸвғЈ Start MongoDB

```bash
# If using local MongoDB
mongod

# If using MongoDB Atlas, ensure connection string is correct in .env
```

#### 2пёҸвғЈ Start Backend Server

```bash
cd backend
npm run dev

# Server will run on http://localhost:5500
# WebSocket server on ws://localhost:5050
```

#### 3пёҸвғЈ Start Frontend

```bash
cd frontend
npm run dev

# Frontend will run on http://localhost:3000
```

#### 4пёҸвғЈ Access the Application

Open your browser and navigate to:
- **Frontend**: http://localhost:3000
- **Backend API**: http://localhost:5500

## рҹ”— Smart Contract Deployment

### Step 1: Get Testnet MATIC

1. Visit [Polygon Faucet](https://faucet.polygon.technology/)
2. Select **Polygon Amoy** network
3. Enter your wallet address
4. Receive free testnet MATIC

### Step 2: Compile Contracts

```bash
cd backend
npx hardhat compile
```

### Step 3: Deploy NGORegistry

```bash
npm run deploy:registry

# OR
npx hardhat run scripts/deployRegistry.js --network amoy
```

### Step 4: Deploy NGOFund

```bash
npm run deploy:fund

# OR
npx hardhat run scripts/deployFund.js --network amoy
```

### Step 5: Update Frontend Environment

Copy the deployed contract addresses from `backend/scripts/deployedAddresses.json` to your frontend `.env.local`:

```env
NEXT_PUBLIC_NGO_REGISTRY_ADDRESS=0xYourDeployedRegistryAddress
NEXT_PUBLIC_NGO_FUND_ADDRESS=0xYourDeployedFundAddress
```

### Testing Smart Contracts

```bash
# Register an NGO
npm run register:ngo

# Make a test donation
npm run donate

# Check NGO balance
npm run balance

# Test withdrawal
npm run withdraw
```

## рҹҡ§ Challenges We Solved

### Technical Challenges

1. **Blockchain Integration Complexity**
   - **Problem**: Integrating smart contracts with backend and frontend
   - **Solution**: Created abstraction layers with Ethers.js and automated event listeners

2. **Simulated vs Real Transactions**
   - **Problem**: Making simulated transactions feel authentic
   - **Solution**: Real testnet deployments with actual transaction hashes

3. **Role-Based Access Control**
   - **Problem**: Managing three different user types with different permissions
   - **Solution**: JWT-based middleware with role verification at multiple levels

4. **Real-Time Updates**
   - **Problem**: Keeping all users informed of transaction status
   - **Solution**: WebSocket integration with event-driven architecture

5. **Gas Fee Management**
   - **Problem**: Testnet transactions failing due to gas estimation
   - **Solution**: Dynamic gas pricing with 2x multipliers for reliability
